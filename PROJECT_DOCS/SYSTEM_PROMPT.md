## 🏗️ Архитектура проекта

### Frontend

**Тип:** Single Page Application (SPA)

**Файлы:**

| Файл | Назначение |
|------|------------|
| `index.html` | Разметка, модальные окна |
| `css/styles.css` | Все стили (1000+ строк) |
| `js/app.js` | Основная логика (рендеринг, события) |
| `js/data.js` | Данные о вкусах Monster Vapor |
| `js/storage.js` | localStorage + Supabase обёртки |
| `js/supabase-config.js` | Прямое подключение к Supabase |

### База данных

**Таблица:** `cards`

**Поля:**
- `card_id` (text, unique) — ID карточки
- `flavor` (text) — Название вкуса
- `rating_sasha`, `rating_nastya` (float8) — Рейтинги
- `emojis` (jsonb) — Массив иконок
- `is_other` (bool) — Вкладка "Эксперименты"

**Важно:** Все данные синхронизируются через Supabase. localStorage используется только как кэш.

## 🔧 Типичные задачи и решения

### Добавление новой функции

1. **Создайте функцию** в соответствующем файле
2. **Добавьте обработчик** в `initApp()`
3. **Протестируйте** локально
4. **Обновите документацию**

### Изменение стилей

1. **Найдите существующий класс** через поиск
2. **Измените или добавьте** свойства
3. **Проверьте** что не сломали другие элементы
4. **Обновите раздел** "Стили и дизайн"

### Работа с базой данных

1. **Откройте Supabase Dashboard**
2. **SQL Editor** → выполните запрос
3. **Обновите `supabase-config.js`** если нужно
4. **Обновите раздел** "База данных"

### Добавление новой иконки

1. **Добавьте в `index.html`** (редактор карточки)
2. **Обновите `data.js`** (`availableEmojis`)
3. **Добавьте приоритет** в `sortEmojisByPriority()`
4. **Обновите документацию**
