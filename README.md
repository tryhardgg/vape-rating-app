## ℹ️ О проекте

**Monster Vapor Tracker** — веб-приложение для совместной оценки жидкостей для вейпа.

**Для кого:** Для парня и девушки (Саша и Настя) чтобы видеть оценки друг друга.

**Технологии:**
- HTML, CSS, JavaScript (ванильный)
- Supabase (база данных + синхронизация)
- GitHub Pages (хостинг)

---

## ✨ Возможности

| Функция | Описание |
|---------|----------|
| **34 карточки Monster Vapor** | Все вкусы 20mg крепости |
| **Рейтинги** | Отдельно Саша + Настя (0-10) |
| **Средний рейтинг** | Автоматический расчёт |
| **Иконки-индикаторы** | 🌪💎⚠️🧼🍯 (5 штук) |
| **Примечания** | Заметки к каждой карточке |
| **Загрузка фото** | Свои изображения жидкостей |
| **Синхронизация** | Данные общие между устройствами |
| **Поиск** | Фильтрация по названию вкуса |
| **Сортировка** | По рейтингу (от высшего к низшему) |
| **Добавление карточек** | Вкладка "Эксперименты" для custom жидкостей |

## 🛠️ Установка и запуск

### Локальный запуск (для разработки)

1. **Склонируйте репозиторий:**
   ```bash
   git clone https://github.com/tryhardgg/vape-rating-app.git
   cd vape-rating-app
   ```

2. **Откройте index.html** в браузере

3. **Или используйте Live Server** в VSCode

### Публикация изменений

```bash
git add .
git commit -m "Описание изменений"
git push origin dev      # Для разработки
git push origin main     # Для публикации (сайт обновится)
```

## 🔀 Работа с Git

### Ветки

| Ветка | Назначение |
|-------|------------|
| **main** | ✅ Стабильная версия (сайт для Насти) |
| **dev** | 🔧 Разработка и тестирование |

### Workflow (как вносить изменения)

```bash
# 1. Переключиться на ветку разработки
git checkout dev

# 2. Внести изменения в код

# 3. Протестировать локально (открыть index.html)

# 4. Закоммитить в dev
git add .
git commit -m "Новая фича"
git push origin dev

# 5. Когда всё готово — влить в main
git checkout main
git merge dev
git push origin main
```

### GitHub Pull Request (опционально)

1. GitHub → Pull requests → New pull request
2. Выберите: `dev` → `main`
3. Опишите изменения
4. Merge pull request

## 🗄️ Supabase (База данных)

### Настройка

1. Проект: [Supabase Dashboard](https://supabase.com/dashboard/project/ajihwuotwwogahsgqdnv)
2. Таблица: `cards`
3. RLS Policy: Разрешить все операции (SELECT, INSERT, UPDATE, DELETE)

### Структура таблицы `cards`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int8 | Первичный ключ (авто) |
| `card_id` | text | ID карточки (уникальный) |
| `flavor` | text | Название вкуса |
| `note` | text | Примечание |
| `image` | text | Изображение (base64 или URL) |
| `rating_sasha` | float8 | Рейтинг Саши (0-10) |
| `rating_nastya` | float8 | Рейтинг Насти (0-10) |
| `emojis` | jsonb | Выбранные иконки (массив) |
| `is_filled` | bool | Заполнена ли карточка |
| `created_at` | timestamptz | Дата создания |
| `updated_at` | timestamptz | Дата обновления |
